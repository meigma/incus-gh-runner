# Technical Notes

- Use hexagonal architecture at all times. Keep business logic isolated from CLI, filesystem, network, storage, and other external adapters.
- Prefer functional testing before calling any feature complete. Unit tests are useful, but they do not prove the tool works the way the design intends.
- Take an agile approach to development. Avoid waterfall: underspecify when useful, prototype early, learn from the result, and refine from working behavior.
- Add concise, identifier-led Godoc comments to every Go type, function, and method, including unexported declarations and test helpers, so IntelliSense exposes intent during review.
- The v1 controller uses `github.com/actions/scaleset v0.4.0` with `github.com/lxc/incus/v7 v7.2.0`, manages one scale set in a preconfigured Incus environment, and mutates only explicitly owned runner instances. The minimum supported server is Incus 7.0; Incus 6 is unsupported. The Go client dependency must remain at v7.2.0 or newer because v7.0.0 failed hosted security analysis with nine unmitigated critical/high CVEs.
- Hot standby is required for v1: target capacity is `min(max_runners, min_runners + TotalAssignedJobs)`, and standby capacity must be JIT-registered, connected, and idle.
- Keep synchronous scale-set callbacks free of Incus I/O. Coalesce demand into a single-owner reconciler and execute Incus work through a bounded worker pool.
- Refresh owned Incus inventory through that bounded worker pool and do not overlap a snapshot with create/delete mutations; startup and periodic inventory must count existing provisioning or busy VMs before scheduling deficits.
- Use one JIT configuration and one job per VM; let the guest power off, collect diagnostics, delete the VM, and reconstruct state from GitHub plus Incus metadata without a v1 database.
- Incus ownership is authorized by the exact `user.incus-gh-runner.owner` value, not by project membership or name prefix. Created VMs also carry correlation, creation-time, and image metadata, and deletion rechecks ownership immediately before mutation.
- The reusable image is optional. mise installs checksum-pinned distrobuilder 3.3.1 from source on Linux; build the reference VM with `mise exec -- image/build.sh`, then validate boot separately on Incus 7 or newer with `image/validate-incus.sh` in an explicitly disposable, non-default Incus project.
- Session 016 evaluated and rejected replacing distrobuilder with bootc. Fedora 44 passed the full Incus guest contract, but required OCI-to-qcow2 packaging, GPT slack, SELinux agent relabeling, writable `/opt` redirection into `/var`, and separate provenance for the derived Incus archive; CentOS Stream 10 lacked the Incus agent's required 9p kernel support. Retain the distrobuilder-based Ubuntu 24.04 plan and do not treat bootc as an open migration thread unless a maintainer explicitly reopens it.
- The guest contract writes root-only `/run/incus-gh-runner/payload.json`, creates `payload.ready` only after the payload is complete, deletes both inputs before starting the unprivileged runner, publishes secret-free status, writes only lifecycle metadata to `ttyS0`, and holds a 30-second live-diagnostic grace before terminal poweroff. The controller captures the live console, stops an owned terminal VM if needed, and then deletes it.
- Run the controller as a foreground systemd service with bounded contexts, reconnect backoff, graceful SIGINT/SIGTERM handling, and systemd restart for irrecoverably wedged operations. The shipped unit uses a dynamic user with root-equivalent `incus-admin` socket access and systemd credentials for the GitHub App key.

Avoid artificially short Incus operation deadlines: an asynchronous create can continue after the client returns a deadline error and leave partial state.

Continue with Phase 4 in `.journal/014/JOB_MACHINE_PROOF_IMPLEMENTATION_PLAN.md` for genuine GitHub proof consumption and bounded evidence; TPM-bound credential validation remains Phase 5.

Keep `PrivateDevices=yes`; this is not TPM-native signing or measured boot. The pinned proof library is `github.com/secure-systems-lab/go-securesystemslib v0.11.0`; retain `actions/scaleset v0.4.0` and Incus SDK `v7.2.0` unless a future phase deliberately revalidates and changes them.
